  <name>Task 1: Create printer_setup.py -- guided auth + config save</name>
  <files>.claude/skills/print/scripts/printer_setup.py</files>
  <action>
Create `printer_setup.py` with argparse CLI. Follow the same script pattern as `makerworld_search.py` (JSON stdout, stderr for debug).

**Subcommand: `setup`** (default, the main flow):
1. Prompt for BambuLab email and password via input()
2. Authenticate using `BambuAuthenticator().login(email, password, lambda: input("Enter 2FA code from email: "))`
3. Get user info via `BambuClient(token=token).get_user_info()` to retrieve UID
4. List printers via `client.get_devices()`
5. If no printers found: print error JSON and exit
6. Always show numbered list of all printers (per locked decision: no default, always show list)
7. User picks by number
8. Save config to `~/.claude/bambu-config.json` with structure:
   ```json
   {
     "region": "global",
     "token": "<jwt>",
     "token_created": "<ISO timestamp>",
     "user_id": "<uid>",
     "printer": {"device_id": "...", "name": "...", "model": "..."},
     "presets": {
       "draft": {"layer_height": 0.28, "infill": 15, "speed": 100},
       "standard": {"layer_height": 0.20, "infill": 20, "speed": 80},
       "quality": {"layer_height": 0.12, "infill": 20, "speed": 60},
       "strong": {"layer_height": 0.20, "infill": 50, "speed": 80}
     }
   }
   ```
9. Set file permissions to 0o600 (contains token)
10. Output JSON: `{"status": "ok", "printer": "...", "message": "..."}`

**Subcommand: `switch-printer`**:
1. Load existing config (must have valid token)
2. Re-list printers using saved token
3. User picks new printer
4. Update config file (preserve token, presets)

**Subcommand: `presets`** (list/add/remove/update):
- `presets list` -- show all presets from config
- `presets set <name> --layer-height 0.20 --infill 30 --speed 80` -- add/update preset
- `presets remove <name>` -- remove preset (refuse to remove last preset)

**Error handling:**
- Catch 401 errors: output `{"status": "error", "error": "Authentication failed. Check credentials.", "reauth": true}`
- Catch connection errors: output `{"status": "error", "error": "Cannot reach BambuLab servers."}`
- If config file exists on `setup`, warn and ask to overwrite

**Dependencies:** `pip install bambu-lab-cloud-api` (check at start, print install instructions if missing)

Make the script executable (chmod +x, add shebang `#!/usr/bin/env python3`).

<task type="auto">
  <name>Task 2: Create printer_control.py -- send, status, and print control</name>
  <files>.claude/skills/print/scripts/printer_control.py</files>
  <action>
Create `printer_control.py` with argparse CLI. JSON stdout, stderr for debug. Loads config from `~/.claude/bambu-config.json` at start (error if missing -- tell user to run setup first).

**Token validation on every run:**
- Check `token_created` timestamp. If older than 20 hours, output warning in JSON: `"token_warning": "Token may be expired. Run setup again if you get auth errors."`
- If any API call returns 401, output: `{"status": "error", "error": "Token expired. Run printer_setup.py setup to re-authenticate.", "reauth": true}`

**Subcommand: `send <file_path>`**:
1. Validate file exists and is a .3mf file
2. Detect if 3MF is sliced: check for gcode files inside the zip (any filename containing 'gcode' in the 3MF archive)
3. If NOT sliced:
   - Find BambuStudio CLI at `/Applications/BambuStudio.app/Contents/MacOS/BambuStudio` or via `which bambu-studio`
   - If BambuStudio not found: output error with install instructions, suggest using `open` subcommand instead
   - Slice using: `subprocess.run([bambu_cli, "--slice", "0", "--export-3mf", sliced_path, input_path])`
   - Sliced output goes to a temp file alongside the original (e.g., `<name>_sliced.3mf`)
4. Upload sliced file via `BambuClient(token=...).upload_file(file_path=...)`
5. Start cloud print via `client.start_cloud_print(device_id=..., filename=...)` with retry (3 attempts, 2s apart per Pitfall 6)
6. Output JSON: `{"status": "ok", "message": "Print started", "file": "...", "printer": "...", "sliced": true/false}`

**Flags for `send`:**
- `--preset <name>`: Apply named preset before slicing (modify 3MF settings or pass to BambuStudio CLI)
- `--no-slice`: Skip slicing, send as-is (for already-sliced files)
- `--dry-run`: Show confirmation summary without actually sending

**Subcommand: `open <file_path>`**:
1. Validate file exists
2. Run `subprocess.run(["open", "-a", "BambuStudio", file_path])`
3. Output JSON: `{"status": "ok", "message": "Opened in BambuStudio"}`

**Subcommand: `status`**:
1. Connect MQTT: `MQTTClient(username=f"u_{config['user_id']}", access_token=config['token'], device_id=config['printer']['device_id'], on_message=handler)`
2. `mqtt.connect(blocking=False)`, `mqtt.request_full_status()`
3. Wait up to 5 seconds for response
4. Disconnect and output JSON with all status fields:
   ```json
   {
     "status": "ok",
     "printer": {
       "state": "RUNNING",
       "progress": 45,
       "eta_minutes": 32,
       "layer": 120,
       "total_layers": 267,
       "nozzle_temp": 220,
       "nozzle_target": 220,
       "bed_temp": 60,
       "bed_target": 60,
       "speed": 100,
       "task_name": "phone-stand.3mf",
       "filament": "PLA",
       "print_error": 0
     }
   }
   ```
5. If state indicates error (print_error != 0 or gcode_state == "FAILED"):
   - Include `"error_details"` field with error code
   - Include `"suggestions"` array with troubleshooting steps (e.g., "Check for filament tangle", "Resume print if error is recoverable", "Cancel and restart if print is damaged")

**Subcommand: `pause`**:
1. Connect MQTT, publish pause command: `{"print": {"sequence_id": "0", "command": "pause", "param": ""}}`
2. Output JSON confirmation

**Subcommand: `resume`**:
1. Connect MQTT, publish resume command
2. Output JSON confirmation

**Subcommand: `cancel`**:
1. Connect MQTT, publish stop command
2. Output JSON confirmation

**Subcommand: `filaments`**:
1. Connect MQTT, get full status
2. Parse AMS data from status (ams array with material type, color, temp ranges)
3. Output JSON: `{"status": "ok", "filaments": [{"slot": 1, "type": "PLA", "color": "#FFFFFF"}, ...]}`

Make the script executable (chmod +x, shebang).
