  <action>
In `extract_model_fields()` (line 229), add dimension extraction after the existing license extraction block (around line 287).

MakerWorld __NEXT_DATA__ search result items may contain dimension/size fields under various possible keys. Try these field names in order:
- `dimensions` (dict with w/h/d or width/height/depth)
- `size` (dict or string)
- `boundingBox` / `bounding_box` (dict with x/y/z)
- `length`, `width`, `height` as separate top-level fields

Add to the model dict:
```python
# Dimensions (if available)
dims = None
for dim_key in ["dimensions", "size", "boundingBox", "bounding_box"]:
    dim_val = item.get(dim_key)
    if isinstance(dim_val, dict):
        w = dim_val.get("width") or dim_val.get("w") or dim_val.get("x")
        h = dim_val.get("height") or dim_val.get("h") or dim_val.get("y")
        d = dim_val.get("depth") or dim_val.get("d") or dim_val.get("z") or dim_val.get("length")
        if w and h:
            dims = {"width": w, "height": h}
            if d:
                dims["depth"] = d
            break
    elif isinstance(dim_val, str) and dim_val:
        dims = {"raw": dim_val}
        break

# Also check top-level width/height/length
if not dims:
    w = item.get("width")
    h = item.get("height")
    if w and h:
        dims = {"width": w, "height": h}
        d = item.get("depth") or item.get("length")
        if d:
            dims["depth"] = d

if dims:
    model["dimensions"] = dims
```

Key points:
- Only add `dimensions` key to model dict if data is actually found (do NOT add empty/null values)
- This is best-effort: MakerWorld search results may not include dimensions at all, which is fine
- The field will simply be absent from JSON output if no dimension data exists
