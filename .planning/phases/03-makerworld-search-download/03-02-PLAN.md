  <name>Task 1: Add intent detection and search/download steps to SKILL.md</name>
  <files>.claude/skills/print/SKILL.md</files>
  <action>
Read the current SKILL.md, then extend it with search capabilities. The changes are:

**1. Update the skill description frontmatter:**
Change the description to include search/download capability:
```
description: >
  Generate 3D printable models from natural language descriptions,
  or search MakerWorld for existing models to download.
  Use when the user wants to 3D print something, create a model,
  design a part, find an existing model, or download from MakerWorld.
  Handles OpenSCAD code generation, preview rendering, 3MF export,
  and MakerWorld search/download.
```

**2. Add a new "Step 0: Detect Intent" section BEFORE Step 1.**

This step determines whether the user wants to search for existing models or generate a new one.

**Search indicators** (trigger search flow): "find", "search", "look for", "download", "get me", "existing", "browse", "MakerWorld", "from makerworld"

**Generate indicators** (trigger generation flow): "make", "create", "generate", "design", "build", "print me"

**Ambiguous** (ask the user): When neither set of indicators is present, or both are present. Ask: "Would you like me to search MakerWorld for an existing model, or generate a custom one from scratch?"

If search intent detected: go to Step S1 (search flow).
If generate intent detected: go to Step 1 (existing generation flow, unchanged).

**3. Add "Step S1: Check Playwright Installation" (search flow only).**

Before searching, verify Playwright is available:

```bash
python3 -c "from playwright.sync_api import sync_playwright; print('OK')" 2>/dev/null
```

If NOT OK:
1. Tell the user: "MakerWorld search requires Playwright (a browser automation tool) which isn't installed yet."
2. Ask: "Would you like me to install it? (`pip install playwright && playwright install chromium`)"
3. If user agrees: run the install commands and verify.
4. If user declines: offer to generate the model from scratch instead (go to Step 1).

**4. Add "Step S2: Search MakerWorld".**

Run the search script:
```bash
python3 ~/.claude/skills/print/scripts/makerworld_search.py search "<user's description>" --limit 3
```

Parse the JSON output.

**If status is "error":**
- Tell the user the search failed and why (from the error field).
- Offer: "Would you like me to try again, or generate a custom model from scratch instead?"
- If generate: go to Step 1.

**If status is "ok" but results are empty:**
- Tell the user: "I couldn't find any matching models on MakerWorld for '<query>'."
- Offer: "Would you like me to generate a custom model for you instead?"
- If yes: go to Step 2 (clarification) with the original description.

**If status is "ok" with results:**
Present results as a numbered list (per locked decision -- 3 results, numbered list format):

```
I found 3 models on MakerWorld:

1. Phone Stand - Adjustable
   Rating: 4.8/5 | Downloads: 12,500
   Thumbnail: https://makerworld.bblmw.com/...
   MakerWorld: https://makerworld.com/en/models/2461740
   >> Recommended: Highest rated with 12k+ downloads

2. Minimal Phone Holder
   Rating: 4.5/5 | Downloads: 8,200
   Thumbnail: https://makerworld.bblmw.com/...
   MakerWorld: https://makerworld.com/en/models/3521890

3. Universal Phone Dock
   Rating: 4.2/5 | Downloads: 3,100
   Thumbnail: https://makerworld.bblmw.com/...
   MakerWorld: https://makerworld.com/en/models/1892345

Which one would you like to download? (or say "generate" to create a custom model instead)
```

The recommended model gets a `>> Recommended: <reason>` line beneath it.

Always ask the user to confirm selection -- never auto-download (per locked decision).

**5. Add "Step S3: Download Selected Model".**

After user picks a model (by number or name):

```bash
python3 ~/.claude/skills/print/scripts/makerworld_search.py download <model_id> --name "<model_name>" --output-dir "$HOME/3d-prints/<model-name-slug>"
```

The folder name is derived from the MakerWorld model name (per locked decision -- not the search query). Slugify: lowercase, replace spaces with hyphens, remove special characters.

Parse the JSON output.

**If status is "error":** Tell user download failed and why. Offer to try a different model or generate instead.

**If status is "ok":** Present a summary:

```
Downloaded to: ~/3d-prints/phone-stand-adjustable/
Files:
  - phone-stand.3mf (3MF)
  - phone-stand-plate2.3mf (3MF)

Source: https://makerworld.com/en/models/2461740
```

Then go to Step 8 (offer to open in BambuStudio). Use the first 3MF file for the BambuStudio open command. If only STL files were downloaded, still offer to open in BambuStudio (it handles STL too).

**6. Add note about downloaded models and Phase 2 features.**

After Step S3 summary, add a note in SKILL.md that for downloaded models:
- Scaling: Since there is no .scad source file, suggest the user resize in BambuStudio directly rather than trying to import into OpenSCAD (per locked decision about scaling downloaded models).
- Print settings (Step 12): Can still be applied if a 3MF was downloaded. For STL-only downloads, tell the user to configure settings in BambuStudio.
- Modification (Step 9): Not applicable to downloaded models (no .scad source). If user asks to modify a downloaded model, explain this and offer to generate a similar model from scratch instead.

**7. Add the script reference to the Reference Files section at the bottom:**

```
- @scripts/makerworld_search.py -- MakerWorld search and download CLI (Playwright-based)
```

**Important: Do NOT modify existing Steps 1-12.** Only add Step 0 before them and Steps S1-S3 as a parallel search flow. The generation flow remains exactly as-is.
