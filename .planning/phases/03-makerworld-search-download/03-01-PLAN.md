  <action>
Create `.claude/skills/print/scripts/` directory and write `makerworld_search.py`.

The script must be a standalone CLI invoked as:
```bash
python3 ~/.claude/skills/print/scripts/makerworld_search.py search "phone stand" --limit 3
python3 ~/.claude/skills/print/scripts/makerworld_search.py download <model_id> --name "Phone Stand" --output-dir ~/3d-prints/phone-stand
```

**Architecture:**
- Use `argparse` with subcommands: `search` and `download`
- All output is JSON to stdout (for skill parsing). Errors/debug go to stderr.
- Use Playwright (Python, sync API) with persistent browser context at `~/.cache/makerworld-browser`
- `headless=True` by default

**Search subcommand (`search`):**
- Takes: `query` (positional), `--limit` (default 3)
- Navigate to `https://makerworld.com/en/search/models?keyword=<query>`
- Wait for page content to load past Cloudflare challenge: use `page.wait_for_selector` with a 30-second timeout. Try `[data-testid='model-card']` first, fall back to waiting for `#__NEXT_DATA__` script tag.
- Extract `__NEXT_DATA__` JSON from the page via `page.query_selector("script#__NEXT_DATA__")` and `json.loads(script.inner_text())`
- Parse search results from the JSON. The exact data path needs runtime discovery -- wrap in try/except and log the actual structure to stderr if parsing fails. Expected fields to extract per model: `id`, `name`, `url` (construct from id), `thumbnail` (image URL), `rating`, `download_count`, `like_count`, `license`, `file_formats` (if available).
- If `__NEXT_DATA__` parsing fails or returns no results, try an alternative approach: use `page.evaluate` to call the internal search API directly from the browser context.
- Limit results to `--limit` count.

**Ranking logic:**
- Calculate combined score: `0.6 * (rating / 5.0) + 0.4 * (download_count / max_downloads_in_results)`
- Sort results by score descending
- Mark the top result with `"recommended": true` and add `"recommendation_reason"` (e.g., "Highest rated with 12k+ downloads")
- All other results get `"recommended": false`

**Search output format (JSON to stdout):**
```json
{
  "status": "ok",
  "results": [
    {
      "id": "2461740",
      "name": "Phone Stand - Adjustable",
      "url": "https://makerworld.com/en/models/2461740",
      "thumbnail": "https://makerworld.bblmw.com/makerworld/model/...",
      "rating": 4.8,
      "download_count": 12500,
      "like_count": 3200,
      "score": 0.95,
      "recommended": true,
      "recommendation_reason": "Highest rated with 12k+ downloads",
      "license": "CC BY-SA 4.0"
    }
  ],
  "query": "phone stand",
  "total_found": 3
}
```

**Error output format (JSON to stdout):**
```json
{
  "status": "error",
  "error": "Cloudflare challenge did not resolve within 30 seconds",
  "query": "phone stand"
}
```

**Download subcommand (`download`):**
- Takes: `model_id` (positional), `--name` (model name for folder), `--output-dir` (explicit output path)
- Navigate to model page: `https://makerworld.com/en/models/<model_id>`
- Wait for content to load past Cloudflare
- Extract `__NEXT_DATA__` from model page to get instance IDs and file info
- For each downloadable instance/file:
  - Try internal API: `page.evaluate` to fetch `/api/v1/design-service/instance/{instanceId}/f3mf?type=download` from browser context
  - If that returns a download URL, download the file using `page.evaluate` + fetch, or navigate to the URL
  - Prefer 3MF files. If no 3MF available, look for STL files.
  - Save files to `--output-dir` (the skill will pass `~/3d-prints/<model-name>/`)
- Download ALL files when multiple exist (plates, variants, parts) per locked decision.

**Download output format (JSON to stdout):**
```json
{
  "status": "ok",
  "model_id": "2461740",
  "model_name": "Phone Stand",
  "output_dir": "/Users/user/3d-prints/phone-stand-adjustable",
  "files": [
    {"name": "phone-stand.3mf", "path": "/Users/user/3d-prints/phone-stand-adjustable/phone-stand.3mf", "format": "3mf"}
  ]
}
```

**Error handling throughout:**
- Wrap all Playwright operations in try/except
- On Cloudflare timeout: return error JSON suggesting user retry (persistent context helps on subsequent attempts)
- On `__NEXT_DATA__` parse failure: log actual structure to stderr, return error JSON
- On download failure: return error JSON with reason
- Always close browser context in finally block
- If Playwright is not installed: catch ImportError early, output `{"status": "error", "error": "playwright_not_installed"}` and exit

**Important implementation notes:**
- Make the script executable (`#!/usr/bin/env python3` shebang)
- URL-encode the search query for the URL
- Add a 2-second delay between page navigations (rate limiting protection per research pitfall 2)
- Use `wait_until="networkidle"` for page.goto calls
- Keep extraction logic in dedicated functions (`extract_next_data`, `parse_search_results`, `parse_model_details`) for easy maintenance when MakerWorld changes their data structure
