  <action>
    1. Scaffold frontend with Vite:
       ```
       npm create vite@latest frontend -- --template react-swc-ts
       cd frontend && npm install
       ```
       Per user decision: use npm (not bun).

    2. Configure `frontend/vite.config.ts` with dev proxy (per discretion -- use Vite proxy):
       - Proxy `/ws` to `ws://localhost:8000` with `ws: true`
       - Proxy `/api` to `http://localhost:8000` with `changeOrigin: true`
       See RESEARCH.md Pattern 4 for exact syntax.

    3. Create `frontend/src/hooks/useWebSocket.ts`:
       - Export `useWebSocket()` hook returning `{ status, sessionId, lastMessage }`
       - `ConnectionStatus` type: 'connecting' | 'connected' | 'reconnecting' | 'disconnected' | 'error' | 'taken_over'
       - On connect: set status 'connected', reset retry counter
       - On message:
         - `type: "connected"` -> store sessionId
         - `type: "ping"` -> respond with `{ type: "pong" }` immediately
         - `type: "session_takeover"` -> set status 'taken_over', do NOT reconnect
         - `type: "idle_warning"` -> set lastMessage (App will display it)
         - `type: "session_expired"` -> set status 'disconnected'
         - All other messages -> set lastMessage
       - On close:
         - If code 4000 (takeover): set status 'taken_over', do NOT reconnect
         - Otherwise: exponential backoff reconnect (base 1s, max 30s, add random jitter 0-1s)
         - Max 10 retries before setting status 'error'
         - Set status 'reconnecting' during retries
       - On error: close the socket (triggers onclose reconnect logic)
       - Cleanup: close WebSocket on unmount
       - WebSocket URL: use relative path `/ws` (Vite proxy handles routing to backend in dev)

    4. Replace `frontend/src/App.tsx` with minimal connection status UI:
       - Import and use `useWebSocket()`
       - Display a centered card showing:
         - Connection status with color indicator (green=connected, yellow=connecting/reconnecting, red=error/disconnected, gray=taken_over)
         - Session ID when connected
         - "Reconnecting..." with retry info when reconnecting
         - "Session taken over by another tab" when taken_over
         - "Cannot connect to server" + manual retry button when error
         - Last message type received (for debugging)
       - Use plain CSS (in App.css) -- no UI library needed for this phase
       - This is a diagnostic/verification UI, not the final dashboard

    5. Clean up Vite boilerplate: remove unused default CSS, assets, and counter code.
