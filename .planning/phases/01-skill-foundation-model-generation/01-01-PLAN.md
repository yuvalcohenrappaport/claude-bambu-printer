Create two reference files that SKILL.md will reference during generation.

**openscad-guide.md** — OpenSCAD code generation patterns and best practices for Claude. Include:

1. **Code structure rules**: Always use named parametric variables at the top (never magic numbers). Always set `$fn` (60 for general, 30 for previews, 100+ for final). Use `module` blocks for distinct components. Center objects at origin, use explicit `translate()`.

2. **Template patterns**: Include 3-4 well-structured parametric examples that demonstrate good OpenSCAD generation:
   - Simple box/container with lid (demonstrates CSG difference, tolerance, parametric dims)
   - Phone/tablet stand (demonstrates angles, structural supports)
   - Cable organizer/holder (demonstrates cylinders, array patterns with `for`)
   - Wall-mount bracket (demonstrates screw holes, structural geometry)
   Each template should have: comment header (description, material, dimensions), parametric variables section, modules, and assembly.

3. **Anti-patterns to avoid**: Non-manifold geometry (always extend cuts by 0.1mm), zero-thickness walls, hardcoded magic numbers, missing $fn, origin offset errors from bad spatial reasoning.

4. **OpenSCAD CLI reference**: Exact commands for preview PNG rendering and 3MF export, including all flags (--imgsize 800,600 --projection p --render --colorscheme DeepOcean --autocenter --viewall for preview; -o model.3mf for export).

5. **Error patterns**: Common OpenSCAD stderr errors and how to fix them (syntax errors, non-manifold warnings, CGAL errors).

**materials.md** — Material-specific design parameters. Include:

1. **PLA**: Min wall 1.2mm, tolerance 0.3mm/side, good for rigid parts/organizers/display items, avoid high-temp/outdoor.
2. **PETG**: Min wall 1.2mm, tolerance 0.35mm/side, good for mechanical/water-resistant, avoid precise dimensional needs.
3. **TPU**: Min wall 1.6mm, tolerance 0.4mm/side, good for flexible items (cases, gaskets, mounts), avoid rigid structural parts.
4. **How material choice affects generated code**: Wall thickness variable, clearance/tolerance variable, $fn recommendations, structural reinforcement needs.
5. **Default recommendation**: PLA unless user specifies otherwise or use case suggests different material.

Create the main skill entry point. This is the most critical file -- it's the prompt that controls Claude's behavior when the user wants to 3D print something.

**Frontmatter** (required for Claude Code skill registration):
```yaml
---
name: print
description: >
  Generate 3D printable models from natural language descriptions.
  Use when the user wants to 3D print something, create a model,
  design a part, or make something for their printer.
  Handles OpenSCAD code generation, preview rendering, and 3MF export.
allowed-tools: Bash, Read, Write, Glob, Grep
---
```

**Skill body** — Structure the instructions in this order:

1. **OpenSCAD Installation Check** (SKIL-02):
   - First action: check if OpenSCAD exists at `/Applications/OpenSCAD.app/Contents/MacOS/OpenSCAD` or via `which openscad`
   - If not found: tell user OpenSCAD is required, ask permission to install via `brew install --cask openscad`
   - If user agrees, run the brew install. If user declines, explain they need to install it manually and stop.
   - Do NOT proceed with any generation until OpenSCAD is confirmed available.

2. **Clarification Step** (GEN-01, GEN-03):
   - ALWAYS ask clarifying questions before generating. Never skip this step.
   - Ask about: (a) exact dimensions or size constraints, (b) what it's for / use case, (c) material preference (PLA/PETG/TPU, default PLA)
   - If description is vague (e.g., "make me something cool"), ask what they need and what they'd use it for. Don't guess.
   - Material choice determines wall thickness and tolerance values per @reference/materials.md

3. **OpenSCAD Code Generation** (GEN-02):
   - Reference @reference/openscad-guide.md for patterns and best practices
   - Reference @reference/materials.md for material-specific parameters
   - Generate well-structured OpenSCAD code with:
     - Comment header: description, material, key dimensions
     - Parametric variables section at top (no magic numbers)
     - Appropriate $fn (60 default)
     - Separate modules for distinct components
     - Material-appropriate wall thickness and tolerances
   - For complex models that may be beyond reliable generation: try anyway but warn the user about potential accuracy issues
   - Show the generated OpenSCAD code to the user (per locked decision)

4. **File Organization** (SKIL-03):
   - Create output directory: `~/3d-prints/<descriptive-name>/` (e.g., `~/3d-prints/phone-stand/`)
   - Use lowercase-hyphenated descriptive names derived from the user's request
   - Save the .scad file as `model.scad` in the output directory

5. **Render and Export Pipeline** (CUST-03):
   - Set `OPENSCAD` variable to the confirmed binary path
   - Step 1: Render preview PNG:
     ```
     "$OPENSCAD" --imgsize 800,600 --projection p --render --colorscheme DeepOcean --autocenter --viewall -o "$OUTPUT_DIR/preview.png" "$OUTPUT_DIR/model.scad" 2>"$OUTPUT_DIR/openscad.log"
     ```
   - Step 2: Export 3MF:
     ```
     "$OPENSCAD" -o "$OUTPUT_DIR/model.3mf" "$OUTPUT_DIR/model.scad" 2>>"$OUTPUT_DIR/openscad.log"
     ```

6. **Error Handling and Retry**:
   - If OpenSCAD returns non-zero exit code, read the stderr log
   - Analyze the error, fix the .scad code, save the fixed version, retry
   - Up to 3 total attempts. On each retry, explain what was wrong and what was fixed.
   - After 3 failures: show the error to the user with the .scad code, suggest they can edit it manually

7. **Output Summary**:
   - After successful render + export, present a summary:
     - Show the preview image (reference the preview.png path)
     - List dimensions (from the parametric variables)
     - State the material and relevant design parameters used
     - Show the full file path to the .3mf file
     - Show the generated OpenSCAD code
   - Then ask: "Want me to open this in BambuStudio?" (per locked decision)
   - If yes: `open -a BambuStudio "$OUTPUT_DIR/model.3mf"`
   - If no: done.

**Important instruction to include in SKILL.md**: "Do NOT skip the clarification step. Do NOT auto-open BambuStudio. Do NOT generate code without asking about dimensions, use case, and material first."
