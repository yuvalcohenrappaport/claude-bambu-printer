## Phase Details

### Phase 5: Backend Foundation
**Goal**: Users can connect to the dashboard and establish a stable, isolated session with Claude Code running behind the scenes
**Depends on**: Phase 4 (v1.0 complete)
**Requirements**: INFR-01
**Success Criteria** (what must be TRUE):
  1. User can open the web app in a browser and see a connected state
  2. Backend spawns a Claude Code subprocess per session and cleans it up on disconnect
  3. WebSocket connection stays alive with heartbeat -- no silent drops after idle time
  4. Multiple browser tabs each get isolated sessions with no state bleed between them

### Phase 6: Chat Interface
**Goal**: Users can have a real-time streaming conversation with Claude Code through the browser
**Depends on**: Phase 5
**Requirements**: CHAT-01, CHAT-02
**Success Criteria** (what must be TRUE):
  1. User can type a message and see Claude's response stream in token-by-token
  2. User can scroll back through all messages in the current session
  3. User can interrupt a long response with a stop button
  4. Chat panel shows clear visual distinction between user messages and AI responses

### Phase 7: MCP Server + 3D Preview
**Goal**: When Claude generates or modifies a model, users see it appear in the 3D viewer automatically
**Depends on**: Phase 6
**Requirements**: INFR-02, VIEW-01, VIEW-02
**Success Criteria** (what must be TRUE):
  1. User can view a 3D model in the browser with orbit, zoom, and pan controls
  2. When user asks Claude to generate a model, the 3D preview updates automatically without manual refresh
  3. Claude Code can push status messages and preview updates to the browser via MCP tools
  4. 3D viewer correctly renders STL files generated by OpenSCAD

### Phase 8: Print Controls + Settings
**Goal**: Users can send models to their printer and adjust key settings before printing, all from the dashboard
**Depends on**: Phase 7
**Requirements**: MNTR-01, INFR-04
**Success Criteria** (what must be TRUE):
  1. User can start a print from the dashboard after a confirmation dialog
  2. User can pause, resume, and cancel an active print from the dashboard
  3. User can view and adjust infill, layer height, and speed in a visual editor before printing
  4. Print actions show clear feedback (success, failure, in-progress states)

### Phase 9: MQTT Printer Monitoring
**Goal**: Users can see their printer's real-time status in the dashboard without switching to Bambu Handy or BambuStudio
**Depends on**: Phase 5 (uses WebSocket infrastructure; independent of Phases 6-8)
**Requirements**: INFR-03, MNTR-02, MNTR-03, MNTR-04, MNTR-05
**Success Criteria** (what must be TRUE):
  1. User can see live print progress as percentage with estimated time remaining
  2. User can see real-time nozzle and bed temperatures updating during a print
  3. User can see current layer number out of total layers
  4. User can see which filament is loaded in the AMS and approximate remaining amount
  5. Status updates arrive within seconds of actual printer state changes

**Goal**: Users can interact with AI responses through clickable actions instead of typing commands
